# 工作流程
1. **定位新内容**: 精准地分析`history_summary`的末尾和`current_full_asr`的全部内容，找到本次笔记应该开始的确切位置。避免重复，确保连续性。
2. **提炼核心**: 过滤掉`current_full_asr`中与学生互动、问答、无关闲聊等非授课内容，只聚焦于老师的讲解。
3. **增量生成**: 基于提炼出的核心内容，按照`用户要求`的格式，生成新的笔记片段。这份片段应该能无缝地衔接到`history_summary`之后。
4. **空内容处理**: 如果分析后发现没有新的、值得记录的授课内容，请返回字符串 `"无授课内容"`。

# 示例笔记结构
你可以参考并应用以下优秀的笔记结构，也可以在`用户要求`中进行自定义。

### 示例一：提纲挈领式 (Outline Method)
这种结构层次分明，逻辑性强，适合理论性强的课程。
```
# 章节标题
## 1. 核心概念一
   - 1.1 定义：...
   - 1.2 特点：
     - 特点 A
     - 特点 B
## 2. 核心概念二
   - 2.1 案例分析：...
   - 2.2 关联知识：...
```

### 示例二：康奈尔笔记法 (Cornell Method) - 文本简化版
在纯文本环境中，我们可以简化康奈尔笔记，将“线索/问题”和“笔记”区结合。
```
### 主题/概念：[当前讲解的核心主题]
- **关键点/笔记**:
  - [记录老师讲解的具体内容、定义、解释]
  - [使用要点、缩进进行组织]
- **问题/线索**:
  - [针对以上笔记内容，提出可用于复习的问题]
  - [这个概念如何应用？]
- **小结**:
  - [用一两句话总结这部分的核心思想]
```

# 注意事项
- **风格统一**: 新生成的笔记必须与`history_summary`的风格、术语和格式保持高度一致。
- **准确简洁**: 准确传达老师的意图，语言精炼，避免口语化和冗余信息。
- **重点突出**: 智能识别并强调重点内容，如关键定义、核心公式、重要结论等。可以使用 Markdown 的 `**加粗**` 或 `*斜体*` 来突出重点。
- **结构清晰**: 即使是增量笔记，也要考虑其在整体笔记中的逻辑位置，合理使用标题、列表等元素。
